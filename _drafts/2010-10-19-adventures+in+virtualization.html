--- layout: post title: Adventures in Virtualization date: '2010-10-19T18:28:00.001-07:00' author: Larry Reid tags: - VirtualBox - Virtualization modified_time: '2010-10-19T18:51:19.178-07:00' blogger_id: tag:blogger.com,1999:blog-5778824359157275227.post-4632042637247160515 --- Scenario: A Windows XP VMware VM used to run iTunes to sync an iPhone, and Quickbooks.

Converted to a KVM qcow2 file -- adventure in itself.

Qemu/KVM doesn't support USB 2, which means it can't support the iPhone. It was fine with Quickbooks.

Here's what I had to do:

  1. mergeide.reg
  2. Convert the file from qcow2 to ovi
  3. Set the network adapter to Intel PRO/1000 MT Desktop (NAT)
  4. Create a new VM in VirtualBox. When you get to the part about defining the storage for the VM, select the ovi file
  5. Repair the installation of Windows XP
  6. Activate XP by phone, since the network is broken. No need to talk to a human. Just use the touch tone phone
  7. Log in
  8. I didn't have a network adapter in Windows so I had no network
  9. Delete the VMware network driver
  10. Install the Guest Additions
  11. Set up a shared folder, meaning you make a folder on the host accessible to the Windows guest
  12. Use a browser on the host to download pro2kxp.exe from the internet to the shared folder. CNET should be a reliable source
  13. Connect to the shared folder in the guest
  14. In the XP guest, run pro2kxp.exe (it's the installer for the driver)
You should now have a workable system.

My Quickbooks installation was broken after this.
